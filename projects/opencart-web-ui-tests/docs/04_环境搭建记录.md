# 环境搭建记录

## 一、运行环境

- 主机系统：Windows
- Linux 环境：WSL2 / Ubuntu
- 浏览器：Linux Google Chrome Headless
- 自动化框架：Selenium + pytest
- 报告工具：Allure
- 版本管理：Git
- GitHub 认证方式：SSH

## 二、本地 OpenCart Docker 尝试记录

最初尝试在 Windows WSL2 + Docker Desktop 中本地搭建 OpenCart，过程中遇到以下问题：

1. Docker 端口发布失败；
2. MySQL 3306 端口映射异常；
3. 数据库表未初始化，出现 `oc_store doesn't exist`；
4. 手动执行 OpenCart CLI 安装后，数据库成功初始化；
5. 数据库初始化成功后，页面仍出现 Twig 模板加载异常；
6. 切换 OpenCart release tag 后，发现缺少 master 分支中的 Makefile / Docker 脚手架。

## 三、最终处理决策

本项目目标是完成 Web UI 自动化回归测试实践，而不是修复 OpenCart 本地部署问题。

因此最终选择 OpenCart 官方 Demo 作为被测系统：

- 测试地址：https://demo.opencart.com
- 优点：无需本地安装，便于快速开展自动化测试实践
- 风险：公开 Demo 环境可能不稳定，因此购物车用例标记为 optional

## 四、关键命令

创建虚拟环境：

```bash
python3 -m venv .venv
激活虚拟环境：

```bash
source .venv/bin/activate
```

安装依赖：

```bash
pip install -r requirements.txt
```

运行核心测试：

```bash
pytest -m "not optional"
```

生成 Allure 结果：

```bash
pytest -m "not optional" --alluredir=reports/allure-results
```

生成 Allure HTML 报告：

```bash
allure generate reports/allure-results -o reports/allure-report --clean
```

通过 SSH 测试 GitHub 连接：

```bash
ssh -T git@github.com
