# OpenCart 电商系统 Web 自动化回归测试计划

## 一、项目背景

本项目基于 OpenCart 官方 Demo 前台进行 Web UI 自动化测试实践，目标是围绕电商系统核心用户路径，完成从测试用例设计、Page Object 封装、pytest 自动化执行到 Allure 报告生成的完整过程。

## 二、被测系统

- 系统名称：TutorialsNinja OpenCart Demo
- 测试地址：https://tutorialsninja.com/demo/
- 测试类型：Web UI 自动化回归测试
- 运行环境：Windows WSL2 / Ubuntu
- 浏览器：Linux Google Chrome Headless
- 自动化语言：Python
- 自动化框架：Selenium + pytest
- 报告工具：Allure

## 三、测试范围

本项目覆盖以下核心功能：

1. 首页访问
2. 登录页表单校验
3. 错误账号登录校验
4. 商品搜索
5. 商品详情页校验
6. 加入购物车
7. 购物车页面校验

## 四、不测试范围

由于使用的是公开 Demo 环境，本项目不覆盖：

1. 真实用户注册
2. 真实订单支付
3. 后台管理系统操作
4. 复杂库存、优惠券、物流等业务规则
5. 性能测试与安全测试

## 五、测试风险

1. 官方 Demo 环境页面结构可能变化，导致元素定位失效（官方demo确实不能用）。
2. 官方 Demo 环境网络不稳定，可能导致页面加载超时（官方demo用不了）。
3. 购物车流程依赖 session，可能在公开环境中出现不稳定。
4. 不同浏览器版本可能导致元素展示略有差异。

## 六、风险处理

1. 使用显式等待，减少页面加载慢导致的失败。
2. 使用 Page Object 模式集中维护元素定位。
3. 将购物车用例标记为 optional / xfail，避免公开 Demo 环境导致核心回归失败。
4. 测试失败时自动截图，便于定位问题。

## 七、交付物

1. 自动化测试代码
2. Page Object 页面对象封装
3. pytest 测试用例
4. Allure 测试报告
5. 测试用例文档
6. 缺陷记录样例
7. 环境搭建记录
8. GitHub README
